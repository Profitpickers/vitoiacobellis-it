# Project Mission

Facilitatore Tagaro is a documentation-first MVP project for a business facilitation platform connected to Vito Iacobellis and the domain `vitoiacobellis.it`.

The first MVP context is commercial facilitation and B2B introductions for Masseria Tagaro S.r.l., an internationally-oriented winery based in Locorotondo, Puglia, Italy.

The mission is to increase credibility, support commercial contact, generate qualified leads, and facilitate appointment or conversation requests while preserving a clear separation between MVP scope and future vision.

# Working Principles

- Documentation first
- MVP first
- No feature creep
- Human approval before major implementation
- Phase-by-phase development
- Maintainability for a No-Code Developer

# Repository Rules

- Do not implement future concepts without approval
- Preserve scalability
- Prefer reusable structures
- Avoid unnecessary complexity
- Update documentation before architecture changes

Future agents should keep the repository understandable, lightweight, and aligned with the current project phase.

# Communication Rules

- Be concise
- Explain decisions
- Prefer business clarity over technical complexity
- Always distinguish MVP from Future Vision

Communication should be professional, direct, and useful for business decision-making.

# Protected Concepts

Future agents must treat the following documents as protected project context:

- `docs/context/AI_PROJECT_CONTEXT.md`
- `docs/prd/PRD.md`
- `docs/prd/BUSINESS_FACILITATION_PLATFORM_PRD_TEMPLATE.md`
- `docs/business/RELATIONSHIP_CAPITAL_SYSTEM.md`
- `docs/future-ideas/BUSINESS_FACILITATION_OS.md`

These concepts may influence future development but must not automatically expand MVP scope.

Future concepts such as relationship capital, business facilitation operating systems, trust scoring, ethical reciprocity, referrals, dashboards, CRM, automation, AI assistants, and Supabase integrations require explicit human approval before becoming implementation scope.
